# A股科技基金截图回测模拟器

一个纯前端基金截图回测工具。用户可以一次上传一张或多张基金 App 净值截图，网页会在浏览器本地使用 OCR 识别日期、净值和日涨幅，用户核对数据后，可以对比买入持有、固定 50% 仓位、旧版规则和新版趋势过滤策略。

所有计算都在浏览器本地完成，截图不会上传到任何服务器。本项目适合部署到 GitHub Pages。

## 功能列表

- 批量上传基金 App 净值截图，支持 PNG、JPG、WebP。
- 多张截图会逐张 OCR，并按日期合并去重，适合同一基金分多屏截图导入。
- 使用 Tesseract.js 在浏览器本地 OCR。
- 使用 canvas 做灰度、增强对比度、2 倍放大和可选二值化预处理。
- 自动解析 `YYYY-MM-DD`、`YYYY/MM/DD`、`YYYY.MM.DD` 等日期格式。
- 可编辑、删除、新增、排序净值行，可按净值重算日涨幅。
- 自动校验日期、净值、日涨幅范围和重复日期。
- 回测 `buy_and_hold`、`fixed_50_percent`、`old_rule_strategy`、`new_rule_strategy`。
- 新版策略支持 MA20、MA60、5 日累计涨幅趋势过滤。
- ECharts 展示资产曲线、回撤曲线、仓位变化、买卖点和市场状态。
- 交易日志记录每次加仓减仓原因。
- Web Worker 参数优化，输出 top 20 参数组合。
- 支持导出净值数据、策略指标、交易日志、优化结果和全部每日记录 CSV。

## 本地运行

```bash
npm install
npm run dev
```

Vite 会输出本地访问地址，通常是：

```text
http://localhost:5173
```

## 构建

```bash
npm run build
```

构建结果会输出到 `dist/`。

## GitHub Pages 部署

当前线上版本使用 GitHub Pages 的 `main /docs` 发布方式，不依赖 GitHub Actions。构建后把 `dist/` 同步到 `docs/`，push 到 `main` 后 GitHub Pages 会自动更新。

首次使用需要在 GitHub 仓库中开启 Pages：

1. 进入仓库 `Settings`。
2. 打开 `Pages`。
3. 在 `Build and deployment` 中选择 `Deploy from a branch`。
4. Branch 选择 `main`，目录选择 `/docs`。
5. 推送代码到 `main` 分支。
6. 等待 Pages 构建完成后访问 Pages 地址。

每次更新页面后执行：

```bash
npm run build
cp -R dist/. docs/
git add .
git commit -m "Update site"
git push
```

项目本地仍保留 `.github/workflows/deploy.yml`，如果你的 GitHub token 拥有 `workflow` 权限，也可以改用 GitHub Actions 自动构建并部署 `dist/`。

当前仓库名按 `fund-screenshot-backtester` 配置，`vite.config.ts` 中：

```ts
base: '/fund-screenshot-backtester/'
```

如果仓库名改了，需要同步修改 `vite.config.ts` 里的 `base`，例如仓库名改为 `my-fund-tool`，则改为：

```ts
base: '/my-fund-tool/'
```

## 截图格式说明

建议截图中包含类似表格：

```text
日期          净值       日涨幅
2026-06-09   1.0667    +4.94%
2026-06-08   1.0165    -4.54%
2026-06-05   1.0648    -2.94%
```

支持以下日期格式：

```text
2026-06-09
2026/06/09
2026.06.09
2026-6-8
```

OCR 识别会受截图清晰度、字体、背景色、压缩质量影响。建议截取净值表格主体区域，避免水印、弹窗和复杂背景。

多图导入说明：

- 一次可以选择多张截图。
- 系统会按照选择顺序逐张识别。
- OCR 文本预览区会显示每张截图的识别文本。
- 净值确认表会把多张截图解析出的日期记录合并，并自动去除重复日期。
- 如果多张截图来自不同基金，不建议直接合并回测；可以先分别上传和回测，后续版本可扩展为多基金并排回测。

## 使用流程

1. 打开网页后点击「选择一张或多张基金 App 净值截图」。
2. 预览无误后点击「开始批量识别」。
3. 等待 OCR 进度完成。
4. 在「OCR 文本预览」中查看原始识别文本，必要时手动修正并重新解析。
5. 在「确认净值数据」表格中核对日期、净值和日涨幅。
6. 可新增行、删除错误行、按日期升序排序、按净值重算日涨幅。
7. 点击「确认数据并开始回测」。
8. 查看策略对比、资产曲线、回撤曲线、仓位曲线、买卖点和交易日志。
9. 点击「开始参数优化」搜索更适合当前历史数据的参数组合。

## 如何解释回测结果

- `总收益`：策略期末资产相对初始资金的收益率。
- `最大回撤`：从历史高点回落的最大幅度，越接近 0 越好。
- `夏普比率`：单位波动带来的收益，越高越好。
- `平均仓位`：策略在回测期间的平均资金暴露。
- `交易次数`：策略触发加仓或减仓的次数。
- `市场状态`：新版策略分为主升浪、震荡、破位；破位时最高仓位会被限制。

## 参数优化

参数优化会搜索以下参数：

- 单日下跌加仓阈值。
- 连跌 2 天累计加仓阈值。
- 连跌 3 天累计加仓阈值。
- 单日上涨减仓阈值。
- 连涨 2 天累计减仓阈值。
- 连涨 3 天累计减仓阈值。
- 初始仓位、最高仓位、最低仓位。

可选择优化目标：

- 最大化 `totalReturn`。
- 最大化 `sharpeRatio`。
- 最大化 `totalReturn / abs(maxDrawdown)`。

优化在 Web Worker 中运行，避免页面卡死。历史数据越多，优化耗时越长。

## 导出结果

页面支持导出：

- OCR 识别后的净值数据 CSV。
- 策略指标 CSV。
- 新版策略交易日志 CSV。
- 参数优化结果 CSV。
- 所有策略每日记录 CSV。

CSV 文件会在浏览器本地生成。

## 实际交易说明

场外基金一般按交易日收盘净值确认。15:00 前申购/赎回通常按当日净值。15:00 后通常按下一交易日净值。本模拟器默认用当天净值成交，是简化假设。实盘会受到确认延迟、费率和净值更新滞后的影响。

## 风险提示

- 本工具只做历史数据回测，不构成投资建议。
- 历史收益不代表未来表现。
- OCR 可能识别错误，用户必须核对净值数据。
- 基金交易存在申购费、赎回费、确认延迟、跟踪误差等实际问题。
- 场外基金可能无法按页面模拟的价格实时交易。
