# A股科技基金截图回测模拟器

一个纯前端基金组合回测工具。用户可以输入多只基金代码，设置“近 N 天”窗口，网页会在浏览器本地获取公开净值、日涨幅并计算区间总涨幅。截图 OCR 仍保留为备用入口：每张截图代表一只基金，用户逐只核对数据后，可以把这些基金作为一个组合一起回测。

所有计算都在浏览器本地完成，截图不会上传到任何服务器。本项目适合部署到 GitHub Pages。

## 功能列表

- 输入基金代码批量导入近期净值，支持自定义近 N 个自然日。
- 自动计算实际交易日数量、起止日期和区间总涨幅。
- 批量上传基金 App 净值截图，支持 PNG、JPG、WebP，作为代码导入的备用方式。
- 多张截图会逐张 OCR；每张截图作为一只基金独立保存净值数据。
- 初始资金会平均分配到所有基金资金槽，最后汇总为组合资产曲线、组合回撤和组合交易日志。
- 使用 Tesseract.js 在浏览器本地 OCR。
- 使用 canvas 做灰度、增强对比度、2 倍放大和可选二值化预处理。
- 自动解析 `YYYY-MM-DD`、`YYYY/MM/DD`、`YYYY.MM.DD` 等日期格式。
- 可编辑、删除、新增、排序净值行，可按净值重算日涨幅。
- 自动校验日期、净值、日涨幅范围和重复日期。
- 回测 `buy_and_hold`、`fixed_50_percent`、`old_rule_strategy`、`new_rule_strategy`。
- 新版策略支持 MA20、MA60、5 日累计涨幅趋势过滤。
- ECharts 展示资产曲线、回撤曲线、仓位变化、买卖点和市场状态。
- 交易日志记录每次加仓减仓原因。
- 组合参数优化已预留位置，后续可按组合总收益、组合夏普和组合收益回撤比搜索参数。
- 支持导出净值数据、策略指标、交易日志和全部每日记录 CSV。

## 基金代码导入

在「基金代码导入」区域输入一个或多个 6 位基金代码，例如：

```text
000001
001052
161725
```

也可以用空格、逗号或换行分隔。然后设置「近 N 天」，例如 30、60、120、250。

说明：

- 「近 N 天」是自然日窗口。
- 基金净值只在交易日更新，所以实际净值记录数通常少于 N。
- 页面会显示实际交易日数量、起止日期和区间总涨幅。
- 区间总涨幅优先使用累计净值计算；没有累计净值时使用单位净值计算。
- 代码导入使用公开基金净值脚本数据，适合科研时形成可复现的数据窗口。

## GitHub Pages 部署

当前线上版本使用 GitHub Pages 的 `main /docs` 发布方式，不依赖 GitHub Actions。构建后把 `dist/` 同步到 `docs/`，push 到 `main` 后 GitHub Pages 会自动更新。

首次使用需要在 GitHub 仓库中开启 Pages：

1. 进入仓库 `Settings`。
2. 打开 `Pages`。
3. 在 `Build and deployment` 中选择 `Deploy from a branch`。
4. Branch 选择 `main`，目录选择 `/docs`。
5. 推送代码到 `main` 分支。
6. 等待 Pages 构建完成后访问 Pages 地址。

每次更新页面后执行：

```bash
npm run build
cp -R dist/. docs/
git add .
git commit -m "Update site"
git push
```

项目本地仍保留 `.github/workflows/deploy.yml`，如果你的 GitHub token 拥有 `workflow` 权限，也可以改用 GitHub Actions 自动构建并部署 `dist/`。

当前仓库名按 `fund-screenshot-backtester` 配置，`vite.config.ts` 中：

```ts
base: '/fund-screenshot-backtester/'
```

如果仓库名改了，需要同步修改 `vite.config.ts` 里的 `base`，例如仓库名改为 `my-fund-tool`，则改为：

```ts
base: '/my-fund-tool/'
```

## 截图格式说明

建议截图中包含类似表格：

```text
日期          净值       日涨幅
2026-06-09   1.0667    +4.94%
2026-06-08   1.0165    -4.54%
2026-06-05   1.0648    -2.94%
```

支持以下日期格式：

```text
2026-06-09
2026/06/09
2026.06.09
2026-6-8
```

OCR 识别会受截图清晰度、字体、背景色、压缩质量影响。建议截取净值表格主体区域，避免水印、弹窗和复杂背景。

多基金导入说明：

- 一次可以选择多张截图。
- 每张截图代表一只基金。
- 系统会按照选择顺序逐张识别。
- 点击截图缩略图可以切换当前正在核对的基金。
- OCR 文本预览区只显示当前选中基金的识别文本。
- 净值确认表只属于当前选中基金，不会跨基金合并日期。
- 点击确认后，系统会把所有已识别基金作为一个组合一起回测。

## 使用流程

1. 在「基金代码导入」中输入多只基金代码。
2. 设置「近 N 天」窗口。
3. 点击「获取净值数据」。
4. 查看每只基金的实际交易日数量和区间总涨幅。
5. 在「确认净值数据」表格中核对当前基金的日期、净值和日涨幅。
6. 可新增行、删除错误行、按日期升序排序、按净值重算日涨幅。
7. 点击「确认全部基金并开始组合回测」。
8. 查看组合策略对比、组合资产曲线、组合回撤曲线、组合仓位曲线、买卖点和交易日志。

截图入口仍可使用：

1. 点击「选择一张或多张基金 App 净值截图」。
2. 每张截图代表一只基金。
3. 点击「开始批量识别」后逐只核对 OCR 结果。

## 如何解释回测结果

- `总收益`：策略期末资产相对初始资金的收益率。
- `最大回撤`：从历史高点回落的最大幅度，越接近 0 越好。
- `夏普比率`：单位波动带来的收益，越高越好。
- `平均仓位`：策略在回测期间的平均资金暴露。
- `交易次数`：策略触发加仓或减仓的次数。
- `市场状态`：新版策略分为主升浪、震荡、破位；破位时最高仓位会被限制。

## 组合参数优化

当前版本已经改为多基金组合回测。参数优化需要按组合总资产、组合回撤和组合夏普重新设计目标函数，避免使用单只基金优化结果误导判断。

后续可扩展搜索：

- 单日下跌加仓阈值。
- 连跌 2 天累计加仓阈值。
- 连跌 3 天累计加仓阈值。
- 单日上涨减仓阈值。
- 连涨 2 天累计减仓阈值。
- 连涨 3 天累计减仓阈值。
- 初始仓位、最高仓位、最低仓位。

建议优化目标：

- 最大化组合 `totalReturn`。
- 最大化组合 `sharpeRatio`。
- 最大化组合 `totalReturn / abs(maxDrawdown)`。

## 导出结果

页面支持导出：

- OCR 识别后的净值数据 CSV。
- 策略指标 CSV。
- 新版策略交易日志 CSV。
- 所有策略每日记录 CSV。

CSV 文件会在浏览器本地生成。

## 实际交易说明

场外基金一般按交易日收盘净值确认。15:00 前申购/赎回通常按当日净值。15:00 后通常按下一交易日净值。本模拟器默认用当天净值成交，是简化假设。实盘会受到确认延迟、费率和净值更新滞后的影响。

## 风险提示

- 本工具只做历史数据回测，不构成投资建议。
- 历史收益不代表未来表现。
- 公开净值数据源和 OCR 都可能存在延迟或错误，用户必须核对净值数据。
- 基金交易存在申购费、赎回费、确认延迟、跟踪误差等实际问题。
- 场外基金可能无法按页面模拟的价格实时交易。
